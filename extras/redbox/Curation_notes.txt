In system.config.json - "curation" section.


Relations:

field (base field name): eg dc:relation.vivo:Dataset.0.dc:identifier

{

        path = [ dc:relation, vivo:dataset ]

            path to the field in the JSON hierarchy

        identifier = [ dc:identifier ]   

            the lookup for the related dataset

        relationship = [ vivo.Relationship, rdf:PlainLiteral ]

            Where to look for the relationship info

        excludeCondition: {
            path: [ "redbox.publish" ],
            value: ""
        }

            - look in "path", and exclude this relation if it has
              the specified value - in this case, it is excluded
              if the record is not published.

        system =  "redbox" (default is "mint");

        optional = if true, don't block curation of this record.

Given that, an analysis of the default curation configuration in
system-config.json

dc:relation.vivo:Dataset.0.dc:identifier

 - related datasets in ReDBox.
 - Excludes if the related dataset not published
 - Does not block curation

dc:relation.vivo:Service.0.dc:identifier:
 
 - Links to services in Mint
 - No exclusion condition
 - Blocks curation

dc:creator.foaf:Person.0.dc:identifier:

 - Link to people in mint
 - Excluded if the dc:identifier is an NLA id (?)
 - Blocks curation

locrel:prc.foaf:Person.0.dc:identifier
 - Primary contact link.
 - Excluded if an NLA id
 - Blocks

swrc:supervisor.foaf:Person.0.dc:identifier
 - Supervisor
 - Excluded if an NLA id
 - Blocks

foaf:Organization.dc:identifier
 - isManagedBy
 - no exclusion
 - blocks

foaf:fundedBy.vivo:Grant.0.dc:identifier
 - isOutputOf grant
 - excluded if redbox:internalGrant == "on"
